## Skill Format

Each skill follows the OpenClaw standard:
```
skills/<name>/
├── SKILL.md          # Main documentation (YAML frontmatter + markdown)
├── scripts/          # Optional execution scripts
├── references/       # Optional reference docs
└── assets/           # Optional assets
```
